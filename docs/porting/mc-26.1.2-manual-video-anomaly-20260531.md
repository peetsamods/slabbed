## Classification

Correction after Julia's follow-up manual observation: the active live symptom is not only the giant rendered-outline anomaly. Lowered slabs can be visible and targetable while behaving like ghost physical geometry that the player can pass through.

The rendered-outline classification below remains valid only for the `frame_020.jpg` visual artifact. It is not sufficient to explain the ghost-lowered-slab physical-presence symptom.

This evidence does not classify as placement authoring failure:

- `placementExpectedDyMismatchRows=0`
- `loweredSideSlabPlacementVanillaDyRows=0`
- every action row marker is `none`

The observed failure lane is visual outline / shape replay parity.

Two shape families appear:

1. Lowered full-block stone at `dy=-0.5` has shifted full-cube collision/outline bounds but empty interaction/raycast replay shape. Representative row `444`: final target `Block{minecraft:stone}` at `20,-58,24`, `finalOutlineReplayHit=hit`, `finalRaycastReplayHit=miss(empty)`, `collisionBounds=min=(0,-0.5,0),max=(1,0.5,1)`, `interactionBounds=empty`.
2. Ordinary dy0 grass/stone also reports empty interaction/raycast replay shape after manual stone placement. Representative rows `9646` and `10334`: full-cube collision/outline bounds, dy0, but `finalRaycastReplayHit=miss(empty)`.

The dy0 family means the recorder's interaction/raycast replay may be stricter than the actual rendered/crosshair path. The live giant outline in `frame_020.jpg` is still real; the current recorder does not explain it because logged `outlineBounds` remain ordinary empty/full/slab/shifted-full bounds.

## Recorder Tooling Update

Implemented after this finding:

- `LevelRenderer.renderHitOutline(...)` now feeds `BlockOutlineRenderState` into the live cursor recorder when `slabbed.liveCursorIntentRecorder=true`.
- Normal gameplay is unchanged when the recorder property is off.
- New artifact: `rendered-outlines.tsv`.
- New `session.jsonl` row type: `rendered_outline`.
- New counters:
  - `renderedOutlineRows`
  - `renderedOutlineLargeBoundsRows`
  - `renderedOutlineReplayBoundsSplitRows`
  - `renderedOutlineTargetSplitRows`
- New rendered-outline fields include rendered outline pos/state, local shape bounds, world bounds, camera-relative bounds, collision/occlusion/interaction render-state bounds, current hit pos/face/vec, cursor row id, cursor target pos/state/vec/face, and cursor replay outline bounds.

Marker meanings:

- `LIVE_RENDERED_OUTLINE_LARGE_BOUNDS`: the actual rendered outline shape has at least one local dimension greater than 2 blocks.
- `LIVE_RENDERED_OUTLINE_REPLAY_BOUNDS_SPLIT`: rendered outline local bounds differ from the last cursor row's replay `outlineBounds`.
- `LIVE_RENDERED_OUTLINE_TARGET_SPLIT`: rendered outline target pos differs from the last cursor row's final hit pos.
